ForePack Sub Sheet

Add your Supabase credentials to go live. Follow the steps below — it takes about 10 minutes.

Step 1 — Create a Supabase Account & Project
  1. Go to supabase.com and click Start your project
  2. Sign up with GitHub or email
  3. Click New project
  4. Name it forepack-subsheet, set a database password, pick a region close to you
  5. Click Create new project and wait about 60 seconds for it to spin up
Step 2 — Create the Database Tables
  1. In your Supabase project, click SQL Editor in the left sidebar
  2. Click New query
  3. Copy and paste the entire block below into the editor
  4. Click the green Run button
  5. You should see "Success. No rows returned"
create table if not exists fp_bets (
  id text primary key,
  data jsonb not null,
  created_at timestamptz default now()
);
create table if not exists fp_games (
  id text primary key,
  data jsonb not null
);
create table if not exists fp_settings (
  id text primary key default 'main',
  data jsonb not null
);
alter table fp_bets enable row level security;
alter table fp_games enable row level security;
alter table fp_settings enable row level security;
create policy "public read bets" on fp_bets for select using (true);
create policy "public insert bets" on fp_bets for insert with check (true);
create policy "public update bets" on fp_bets for update using (true);
create policy "public delete bets" on fp_bets for delete using (true);
create policy "public read games" on fp_games for select using (true);
create policy "public write games" on fp_games for all using (true);
create policy "public read settings" on fp_settings for select using (true);
create policy "public write settings" on fp_settings for all using (true);
Step 3 — Copy Your Credentials
  1. Click Project Settings (gear icon ⚙️ in the left sidebar)
  2. Click API in the submenu
  3. Under Project URL — copy that URL
  4. Under Project API keys — copy the anon / public key
Step 4 — Paste Credentials Into This File
  1. Open forepack-subsheet.html in Notepad (Windows) or TextEdit (Mac)
  2. Find the section near the top that says CONFIG — PASTE YOUR SUPABASE CREDENTIALS HERE
  3. Replace YOUR_SUPABASE_URL with your Project URL
  4. Replace YOUR_SUPABASE_ANON_KEY with your anon key
  5. Save the file
// Before:
const SUPABASE_URL = "YOUR_SUPABASE_URL";
const SUPABASE_KEY = "YOUR_SUPABASE_ANON_KEY";
// After:
const SUPABASE_URL = "https://xyzxyz.supabase.co";
const SUPABASE_KEY = "eyJhbGciOiJIUzI1NiIsInR5cCI6...";
Step 5 — Deploy to Netlify
  1. Go to app.netlify.com/drop
  2. Drag and drop your saved forepack-subsheet.html file onto the page
  3. Netlify gives you a live URL instantly — share it with your community!
  4. Optional: Click Site configuration → Change site name to get a custom URL like forepack.netlify.app
✅ That's it — you're live!

All data is saved to your Supabase database in real time. Anyone with your Netlify link can view the dashboard and submit picks. Only people with your Level 2 or Level 3 passwords can edit anything.
